# 壶流论文图库原型设计

## 目标

第一版先服务论文写作，而不是做成完整商业软件。核心目标是把 `codex整理` 中的图片整理成可审阅、可筛选、可继续扩展的论文图库原型。

这个原型需要做到：

- 能舒服地看图：缩略图、大图、路径、基础信息一屏可见。
- 能服务论文：主入口按开题报告中的历史分期组织。
- 能沉淀资料：一件器物一个稳定临时编号，多张图片可以归到同一器物下。
- 能继续生长：后续可以增加形制谱系视图、修复标准库视图、手机端界面。
- 不破坏原始材料：不移动、不改名、不删除原图；删除本地文件前必须先征得用户许可。

## 扫描范围

第一版只扫描：

`F:\我爱的\美术\研究生\毕业论文毕业设计\1形制图谱\codex整理`

不扫描同级原始分类目录：

- `01_按时期分类`
- `02_按窑口分类`
- `03_按形态分类`
- `04_修复案例库`
- `99_待分类`

这些目录暂时只作为背景材料，不进入第一版索引。

## 第一版主结构

主入口采用论文章节型分期，贴合开题报告中的第二章结构。

1. 元：起源阶段，史前至夏商
2. 亨：多元阶段，两周至秦汉
3. 利：成熟阶段，魏晋至隋唐
4. 贞：鼎盛阶段，宋元至明清
5. 近现代与补充参照

每个分期下展示该阶段相关器物条目。条目可以继续带有窑口、器类、壶流形态、来源、资料性质等字段。

## 条目模型

软件不把每张图片都当成独立研究对象，而是区分“器物条目”和“图片文件”。

### 器物条目

一件器物或一个明确的残件对应一个条目，例如：

- `HL-0001 河姆渡文化鸟形盉`
- `HL-0002 宋-大余滩头窑-执壶流残件`

条目字段：

- 临时编号
- 条目名称
- 论文分期
- 时代
- 器类
- 窑口或文化类型
- 壶流形态
- 资料性质
- 是否陶瓷壶流本体
- 与壶流关系
- 来源或馆藏
- 审阅状态
- 备注

### 图片文件

一件器物可以挂多张图片，例如正面、侧面、局部、不同来源图、修复前后图。

图片字段：

- 文件路径
- 文件名
- 文件指纹或哈希
- 图片尺寸
- 所属器物编号
- 图片角色：整体、局部、流部特写、来源截图、修复前、修复中、修复后、待判断
- 文件状态：正常、缺失、疑似移动、待导入

## 前身类和参照类

青铜盉、爵、陶鬶、早期陶盉等资料收入图库，但不和陶瓷壶流本体混成一类。

建议字段：

- `资料性质：前身/源流参照`
- `是否陶瓷壶流本体：否`
- `与壶流关系：流部起源、功能参照、形态参照、礼器影响、器型过渡`

这样它们可以服务论文的源流论述，但不会干扰后续陶瓷壶流形制分类。

## 稳定编号规则

第一版自动生成临时编号，格式为：

`HL-0001`

编号稳定绑定到“器物条目”，不是绑定到文件夹顺序。后续新增图片、移动图片、删除图片时，既有器物编号不重排。

新增图片时：

- 重新扫描 `codex整理`。
- 新图片进入 `待导入`。
- 如果文件名像已有条目，进入 `疑似同一器物`。
- 用户确认后，图片挂到已有器物编号下。

删除或移动图片时：

- 软件不删除数据库条目。
- 找不到原路径时，图片标记为 `文件缺失`。
- 如果能通过文件指纹找到新位置，则更新路径。

## 同一器物多图判断

第一版采用“自动候选 + 人工确认”的方式。

自动候选依据：

- 文件名去掉 `(1)`、`(2)`、`-2` 等序号后高度相同。
- 同一路径下名称相近。
- 文件哈希完全相同则标记为完全重复，但不自动删除。

已经发现的典型候选组包括：

- `河姆渡文化鸟形盉 (1)-(4)`
- `河姆渡文化 龟形刻纹陶盉 ... (1)-(3)`
- `宋-大余滩头窑-执壶流残件 (1)-(4)`
- `良渚文化宽把壶 (1)-(2)`
- `良渚文化红陶盉`
- `龙首青铜盉`
- `宋-景德镇湖田窑-影青温酒执壶`

拿不准的组不自动合并，保留为 `待确认合并`。

## 第一版界面

电脑端优先，手机端结构预留。

电脑端布局：

- 左侧：论文分期导航和文件夹来源树
- 中间：图片缩略图网格
- 右侧：大图预览、器物条目卡、图片列表、路径、备注
- 顶部：搜索、筛选、视图切换

基础看图功能：

- 缩略图浏览
- 单图大图预览
- 上一张/下一张
- 按论文分期筛选
- 按器类、时代、资料性质筛选
- 查看原始路径
- 显示同一器物下的多张图

## 数据文件

第一版可先使用本地 JSON 文件，后续再升级到 SQLite。

建议目录：

- `data/catalog.json`：器物条目、图片、标签、状态
- `data/import-log.json`：每次扫描结果
- `data/pending-merge.json`：待确认合并组
- `prototype/`：本地网页原型代码

## 后续扩展

第一版完成后，可以逐步增加：

1. 形制谱系视图：按直流、曲流、鸭嘴流、鸡首流、长流、军持流等组织。
2. 修复标准库视图：按完整参照器、残件、流部局部、修复阶段组织。
3. 对比模式：并排比较不同时期、不同窑口、不同流型。
4. 导出功能：导出论文图录清单、附录年表、待补来源清单。
5. 手机端 PWA：用手机浏览、标记、查图。

## 第一版完成标准

- 成功只读扫描 `codex整理`。
- 自动生成器物条目和图片索引。
- 自动识别一批疑似同一器物多图候选。
- 能在浏览器中按论文分期看图。
- 能查看大图、路径、编号、基础字段。
- 不修改、不移动、不删除任何原始图片。
